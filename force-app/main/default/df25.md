    Problem with RunSpecifiedTests
        1. Ownership over DevOps Pipeline
        2. Strict class to test naming patterns
        3. Parse Commit Messages for Test or PR Body
        4. Educate Developers
        5. How about triggers
        6. .........

    Solution - RunReleventTests

        1. Salesforce will auto ditermine and execute the subset of the test which are relevant to deployment components
        2. Improve the developer experience with faster feedback loop and deployment time
        3. Test will scale with the size of the deployment payload
        
        @IsTest(testFor="ApexClass:AccountHandler,ApexTrigger:AccountTrigger")
        public with sharing class AccountTriggerHandler{
            // only runs if the deployment components either AccountHandler Apex class or AccountTrigger Apex Trigger
        }

        @IsTest(critical=true)
        public with sharing class AccountTriggerHandler{
            // Always runs regardless of the deployment components
        }

        How this works as a Whole
            1. Tests in the deployment metadata components
            2. Tests that Salesforce discovers 
            3. Tests that are marked as testFor
            4. Tests that are marked as critical
